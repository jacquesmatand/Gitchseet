Commandes

Définir le nom que vous voulez associer à toutes vos opérations de commit. La configuration peut aussi se faire avec son adresse mail avec la commande $ git config --global user.email "[adresse email]"

$ git config --global user.name "[nom]"

Pour démarrer un projet, il faut créer un repository. Grâce à la commande giti init, on l'initialise pour la suite des tâches. L'on peutt soit créer le projet sur le site Github et sur un terminal qui sera mis en lien avec Github. R

$ git init [nom-du-projet]

Un fichier, dossier, une fois créé doit être ajouté au dossier principal pour être reconnu et pris en compte par git. Et ce grâce à la commande suivante. Elle doit se faire à chacune des modifications.

$ git add

Un projet git dispose d'un statut. Pour connaître l'état ou le statut, la commande suivante permet de le savoir. Cette commande permet aussi de voir les différentes modifications du document( les commit)

$ git status

Les différents commit, une fois enregistrés, permettent de suivre l'évolution du projet, les différentes successives et facilitent le versionning. Aussi, grâce aux commits on sait voir ce qui a été fait, quand, par qui et pourquoi. D'où l'importance du message ( -m) descriptif et explicatif du commit.

$ git commit -m "[message descriptif]"

Il permet de voir l'historique du projet et les différentes modifications faites dessus.

$ git log
